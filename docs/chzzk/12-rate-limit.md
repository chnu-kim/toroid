# Rate Limit


> API 호출 제한, 429 응답 처리, 앱 자동 삭제 정책을 정리한 문서입니다.

---


## Rate Limit 정책


Chzzk API는 과도한 호출을 방지하기 위해 Rate Limit을 적용합니다. 제한 초과 시 `429 TOO_MANY_REQUESTS` 응답을 반환합니다.

```json
{
  "code": 429,
  "message": "TOO_MANY_REQUESTS"
}
```

---


## 429 응답 처리


Rate Limit 초과 시 **Exponential Backoff** 전략을 권장합니다.

```
1차 재시도: 1초 대기
2차 재시도: 2초 대기
3차 재시도: 4초 대기
4차 재시도: 8초 대기
...
```

### 구현 가이드

1. `429` 응답 수신 시 즉시 재시도하지 않습니다.
2. 대기 시간을 지수적으로 증가시킵니다.
3. 최대 재시도 횟수를 설정합니다.
4. Jitter(임의 지연)를 추가하여 동시 재시도를 분산합니다.

---


## 앱 자동 삭제 정책

- **90일**간 API Scope 사용이 없는 앱은 자동 삭제됩니다.
- 삭제 전 개발자 센터를 통해 알림이 전송됩니다.
- 삭제된 앱의 `clientId`, `clientSecret`은 더 이상 유효하지 않습니다.

---


## 세션 연결 제한


실시간 이벤트 세션에도 연결 수 제한이 있습니다.

| 인증 방식             | 최대 동시 연결 | 세션당 최대 이벤트 구독 |
|-------------------|----------|---------------|
| Client 인증         | 10개      | 30개           |
| User Access Token | 3개       | 30개           |
